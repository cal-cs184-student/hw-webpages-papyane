Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau’s algorithm defines a point on the Bezier curve between two points based on the recursive linear interpolation of n control points. To implement it, we take the given set of n points in evaluateStep, for every pair of adjacent points we linearly interpolate between the two points to create a new point p_i. This gives us a total of n-1 intermediate points when calling evaluateStep.

Part 2: Bezier surfaces with separable 1D de Casteljau

To evaluate a Bezier surface, we use de Casteljau’s algorithm to break down a 2d surface evaluation into numerous 1D curve evaluations. We defined a Bezier surface with a nxn grid of control points and to find a point on this surface at some (u,v), we repeatedly use de Casteljau’s algorithm. Each row is treated as an independent Bezier curve, so we use 1D de Casteljau’s with u as the parameter. Then we get n intermediate points and run 1D de Casteljau’s again but using v as the parameter. This gives us the point on our surface. evaluateStep performs the basic single iteration of de Casteljau’s. Evaluate 1D completes the entire run of de Casteljau’s to a single point. Evaluate brings implements the algorithm described above. This method works because the algebraic formula for a Bézier surface is a product of two Bernstein polynomials (one for u and one for v). Because these terms are multiplied, you can group the u terms together and evaluate them first, effectively reducing a 2D problem into a series of 1D problems The extension of de Casteljau's algorithm to surfaces relies on the property of separability, which allows us to 2D surface evaluation to be broken down.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

It is necessary to create the normal of a vertex through the averaging of the normals of its surrounding faces in order to correctly implement Phong shading. To implement this, we iterate through the faces of a vertex through the half_edges. At each face, we extract the three vertices, p0 p1 p2 that it corresponds to using h->vertex->position, h->next->vertex->position and so on. Then we calculate the cross product between the vectors defined by p1 - p0 and p2 - p0, adding this to our sum which will return later (make it a unit vector before returning of course). First, we can use p1 - p0 and p2 - p0 in that order because we are always following CCW across a face by nature of halfedges, thus we will always have an outward facing normal. Second, when we calculate this cross product, it is already weighted by area! Because the magnitude of the cross product of these two vectors will give us the area of the parallelogram defined by the two vectors. Meaning that the vector normal will be weighted by area through the nature of calculating the cross product.

Part 6: Loop subdivision for mesh upsampling

To do loop subdivision, we split it up into five main steps. First we compute the position of the new vertices by simply piecing together the formula (getting the sum of the positions of neighbors of v, then calculating based on the degree of v). We store this in newPosition. Second, we calculate where the new vertices will sit between edges. The next step is to split every edge in the mesh. We put all the old edges into a separate array (or vector more precisely), and then begin to split those. This separation is necessary in order for us not to split edges we have just split. Then, we flip any new edges that connect old vertices to new vertices. Finally, we update the positions of every vertex with their new position. After observing what happens when we loop subdivide, we can see that subdivision acts as a geometric low-pass filter. It rounds off sharp corners and edges because the weighted averaging math pulls vertices toward the center of their local neighborhood. We can mitigate this effect by pre-splitting edges to have the averaging math work over a smaller area, which pins the sharp edges in place more so than before. The cube’s asymmetry is a consequence of directional bias in the initial triangulation, as each face is split by a single diagonal, we’ll have three incident edges while others will have six. The subdivision math will tug more aggressively on the vertices with more incident edges. We preprocess the cube to have a symmetric X pattern on every face through splitting. Then the edges that are connected to every vertex are the same number for all of them. Equalizing this and centering the geometry allows the subdivision weights to be applied uniformly, which gives us a more balanced symmetrical shape.
